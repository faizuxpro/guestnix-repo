# Airbnb Guest Messaging Playbook: Lifecycle Templates, Escalation Logic, and KPI Control

High-performing guest messaging is a designed system, not a set of random scripts. The goal is simple: reduce uncertainty at every stay milestone while preserving a human handoff path for high-risk scenarios. Hosts who implement lifecycle messaging with clear escalation rules improve review quality and lower support load.

## Quick Answer
Build messaging around the guest journey:
- Pre-booking qualification.
- Booking confirmation.
- Pre-arrival instructions.
- Arrival support.
- Mid-stay health check.
- Checkout reminder.
- Post-stay follow-up.

Each message should have one objective, one clear action, and one link to the exact guidebook section.

## Messaging Architecture (The Right Way)
Use this structure:
- `Trigger`: when message sends.
- `Intent`: what confusion it prevents.
- `Primary CTA`: what guest should do next.
- `Fallback`: what happens if guest is stuck.
- `Escalation`: who handles high-risk replies.

## Lifecycle Trigger Map
| Stage | Trigger | Objective | Main CTA | Escalation condition |
|---|---|---|---|---|
| Pre-booking | Inquiry received | Qualify fit and build confidence | Confirm dates + constraints | Policy conflict |
| Booking confirmed | Reservation created | Set expectations | Confirm arrival details | Special request complexity |
| 72h pre-arrival | Time-based automation | Reduce check-in anxiety | Review check-in section | Access concern |
| Day of arrival | Check-in window start | Prevent entry friction | Open entry instructions | Cannot access property |
| Mid-stay | Morning after first night | Catch issues early | Report any comfort issues | Safety/utility complaint |
| Evening before checkout | Time-based automation | Reduce checkout confusion | Follow checkout checklist | Extension request conflict |
| 12-24h post-stay | Stay completed | Capture feedback/review | Leave review | Complaint raised post-stay |

## Core Templates (Advanced, Not Generic)

## 1) Pre-Booking Qualification
```text
Hi [Guest Name], thanks for your message.

Quick fit check for your stay:
- Number of guests: [confirm]
- Arrival time window: [confirm]
- Any special requirements: [confirm]

Key house points:
- Quiet hours: [time range]
- Parking: [rule]
- Check-in type: [self/manual]

If this works for you, I can confirm next steps right away.
```

## 2) Pre-Arrival Instruction
```text
Hi [Guest Name], your arrival is coming up.

Please open your stay guide here: [Guidebook Link]
Start with:
1) Entry instructions
2) Parking
3) Wi-Fi
4) House rules

If you hit any issue on arrival, send a message with a photo and I will assist immediately.
```

## 3) Arrival Friction Recovery
```text
I see you are having trouble entering. Let's fix this quickly.

Step 1: Please try [backup method].
Step 2: If it fails, send a photo of [specific item].
Step 3: I will activate backup access right away.
```

## 4) Mid-Stay Issue Detection
```text
Quick check-in: is everything working as expected in the home?

If anything needs attention, tell me now so I can resolve it during your stay.
```

## 5) Checkout Reminder
```text
Friendly reminder: checkout is at [time].

Before leaving:
1) Place trash in [location]
2) Lock all entry points
3) Send me a quick "checked out" message
```

## 6) Post-Stay Review Prompt
```text
Thank you for staying with us, [Guest Name].

I appreciate your stay and would value your review.
If anything fell short, reply here and I will address it directly.
```

## Tone System: Professional, Warm, Precise
Tone rules:
- Use plain language.
- Do not over-apologize before diagnosing.
- Avoid legalistic tone in normal flows.
- Be explicit in policy enforcement messages.

Message quality test:
- Can a guest understand the action in 5 seconds?
- Is there a backup path?
- Is escalation clearly implied where needed?

## Difficult Scenario Scripts

## Early Check-In Denial (Without Friction)
```text
I understand you'd like early check-in.
I cannot confirm that yet because turnover is still in progress.
I will update you by [time] as soon as the home is ready.
```

## Noise Warning
```text
We received a noise report from nearby residents.
Please reduce volume immediately to comply with quiet-hour policy.
If noise continues, we may need to escalate per booking rules.
```

## Unauthorized Visitor Notice
```text
Our booking allows only registered guests.
Please confirm current occupancy now.
Additional visitors require prior approval.
```

## Automation Rules (What to Automate vs Not)
Automate:
- Pre-arrival reminders.
- Guidebook links.
- Checkout reminders.
- Post-stay thank-you.

Do not fully automate:
- Safety incidents.
- Payment disputes.
- Rule violations with potential cancellation.
- Guest dissatisfaction requiring compensation decisions.

## Internal Routing and Ownership
Assign clear ownership:
- `L1`: frontline template-based support.
- `L2`: operational troubleshooting.
- `L3`: host final decisions (refunds, policy enforcement).

Tag every conversation:
- `prebooking`, `arrival`, `access`, `comfort`, `policy`, `checkout`, `review`.

This enables clean analytics and process improvement.

## KPI Dashboard for Messaging Quality
| KPI | Target | Why it matters |
|---|---|---|
| First response median | < 15 min | Perceived service quality |
| Resolution time (non-urgent) | < 2 hr | Support efficiency |
| Arrival issue rate | < 5% | Instruction quality |
| Messages per booking | Downtrend over time | Self-serve effectiveness |
| Communication-positive review mentions | Uptrend | Guest satisfaction signal |

## Experiment Framework

## FAQ

## How many messages are ideal per stay?
Enough to prevent confusion, but not so many that guests ignore them. Focus on milestone triggers.

## Should every message include the full guidebook link?
Use deep links to relevant sections where possible. Full link can appear in key lifecycle messages.

## Can AI handle all communication?
AI can handle routine Q&A and drafting. Human oversight is required for risk, policy, and compensation decisions.

## What is the best first improvement for most hosts?
Rewrite pre-arrival and arrival-day messages with clearer action steps and backup paths.

## How do I reduce repetitive questions fastest?
Link each high-frequency question to one definitive guidebook section and reuse that flow consistently.
